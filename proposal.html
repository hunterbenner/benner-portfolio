Portfolio Website Proposal

Hunter Benner — CTS1831C Web Design

Website Purpose

The purpose of this portfolio website is to show employers and recruiters what I can do as a software developer. Instead of just handing in a resume, this site lets me show actual projects I have built, list my skills, and make it easy for someone to contact me. It will be my main online presence when I start looking for jobs.

Target Audience

The people I am building this site for are:

  • Employers and hiring managers looking for software developers
  • Recruiters searching for entry-level candidates
  • Clients who need a developer for freelance projects
  • Other developers interested in collaborating

Career Goals

My goal is to get a job as a junior software developer after I finish my degree. I am most interested in front-end and full-stack web development. Down the road I want to grow into a senior developer role and work on projects that actually make a difference. This portfolio is how I start putting myself out there.

Site Plan

My website will have at least five pages:

  • Home (index.html) - Introduction and a quick summary about me
  • Projects (projects.html) - Projects I have built with descriptions and links
  • Skills (skills.html) - Programming languages and tools I know how to use
  • Resume (resume.html) - My work experience, education, and a downloadable resume
  • Contact (contact.html) - How to reach me, links to GitHub and email
  • Proposal (proposal.html) - This page, not included in the navigation

File Architecture

The root folder is named hunterbenner.github.io and contains these subfolders:

  • css - Holds the stylesheet files that control how the site looks
  • assets - Holds general files like fonts or downloadable documents
  • images - Holds all the image files used on the site
  • media - Holds any video or audio files
  • scripts - Holds JavaScript files that add interactivity
  • js - Holds additional JavaScript libraries

All folder and file names are lowercase with no spaces or special characters.

Portfolio Components for Employment

After researching what employers look for, I found that a strong software developer portfolio needs to include these things:

  • Real projects with live links - Employers want to see working code, not just a list of things you claim to know
  • Skills section - A clear list of languages and tools like HTML, CSS, JavaScript, Python, and Git
  • About me section - A short bio that explains who you are and what kind of work you are looking for
  • Downloadable resume - A PDF version of your resume that is easy to save and share
  • Contact information - Email, GitHub link, and a simple contact form so employers can reach you
  • Mobile friendly design - The site needs to look good on phones and tablets, not just computers
  • Clean professional design - Simple layout, readable fonts, and consistent colors show that you pay attention to detail
